# Implementation Plan: Fix Document Search Returns No Results

**Branch**: `015-fix-document-search` | **Date**: 2026-06-20 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/015-fix-document-search/spec.md`

## Summary

Search always returns zero results because three compounding bugs in the indexing pipeline prevent chunks from ever being written to the database with embeddings. Fix all three bugs and add a "no results" message to the frontend. No schema migrations are needed.

## Technical Context

**Language/Version**: Python 3.12 (API + Workers), TypeScript / Next.js (frontend)

**Primary Dependencies**: FastAPI, SQLAlchemy 2.0 async, pgvector, Celery, Ollama (embedding), React

**Storage**: PostgreSQL with pgvector extension — `chunks` table (768-dim vector column)

**Testing**: pytest + pytest-anyio (API/Workers), Vitest (frontend)

**Target Platform**: Linux server (Docker/Kubernetes)

**Project Type**: Web application with background workers

**Performance Goals**: Search results within 2 seconds (SC-002)

**Constraints**: Bug fix only — no new infrastructure, no schema changes

**Scale/Scope**: Fix restores existing functionality; no scaling changes needed

## Implementation Phases

### Phase 1: Fix `upsert_chunks` (Bugs 1 + 2)

**File**: `apps/api/tessera_api/adapters/repo.py`

**Change**: Replace the broken `upsert_chunks` method body with raw SQL that mirrors the existing `search()` and `delete_by_document()` pattern:

```python
async def upsert_chunks(self, chunks: list[Chunk]) -> None:
    from sqlalchemy import text
    for chunk in chunks:
        await self._session.execute(
            text("""
                INSERT INTO chunks
                    (id, document_version_id, document_id, space_id,
                     ordinal, text, embedding, confidentiality, language)
                VALUES
                    (:id, :document_version_id, :document_id, :space_id,
                     :ordinal, :text, CAST(:embedding AS vector), :confidentiality, :language)
                ON CONFLICT (id) DO UPDATE SET
                    text = EXCLUDED.text,
                    embedding = EXCLUDED.embedding,
                    ordinal = EXCLUDED.ordinal
            """),
            {
                "id": str(chunk.id),
                "document_version_id": str(chunk.document_version_id),
                "document_id": str(chunk.document_id),
                "space_id": str(chunk.space_id),
                "ordinal": chunk.ordinal,
                "text": chunk.text,
                "embedding": str(chunk.embedding) if chunk.embedding else None,
                "confidentiality": chunk.confidentiality.value,
                "language": chunk.language,
            },
        )
```

**Tests** (`apps/api/tests/unit/test_chunk_repository.py` — new file):
- `test_upsert_chunks_executes_insert_for_each_chunk` — verify `session.execute` is called once per chunk
- `test_upsert_chunks_includes_embedding_in_params` — verify `embedding` key is present in params
- `test_upsert_chunks_skips_null_embedding_gracefully` — chunk with `embedding=None` must not raise

### Phase 3: Prepend title to chunk content

**File**: `apps/workers/tessera_workers/indexing/_index.py`

**Change**: Before calling `chunk_document`, prepend the document title to the version's content:

```python

    update={"content_markdown": title_prefix + version.content_markdown}

    version=version_with_title,
    document_id=document_id,
    space_id=space_id,
    confidentiality=document.confidentiality,
)
```
