# Data Model: Fix Chat Submit Crash on UUID Generation

**Feature**: 025-fix-crypto-uuid

---

## No Persistent Data Model Changes

This fix is entirely client-side. The `turnId` produced by `generateId()` is ephemeral React state — it lives only in the `turns` array within `ChatInterface` for the duration of a browser session and is never sent to the server, stored in a database, or persisted in any way.

The existing `ChatTurn` type (in `lib/types.ts`) is unchanged:

```typescript
export interface ChatTurn {
  id: string;          // generated by generateId() — ephemeral, session-scoped
  question: string;
  answer: AnswerResponse | null;
  status: "pending" | "complete" | "error";
  errorMessage?: string;
}
```

## New Utility Entity

The only addition is a pure function in the utility layer:

| Artifact | Location | Purpose |
|---|---|---|
| `generateId` | `apps/web/lib/utils/generate-id.ts` | Produces a UUID v4 string that works in all browser contexts |

**Invariants**:
- Output MUST conform to UUID v4 format (`xxxxxxxx-xxxx-4xxx-yxxx-xxxxxxxxxxxx`).
- Two successive calls MUST NOT return the same value.
- No side effects; no I/O; pure function.
