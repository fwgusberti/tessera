# Implementation Plan: Fix Publish — Record Approval on Existing Version

**Branch**: `013-fix-publish-version-update` | **Date**: 2026-06-20 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/013-fix-publish-version-update/spec.md`

## Summary

`POST /documents/{id}/publish` returns 500 because it calls `ver_repo.create(approved_version)` with a `model_copy` of the latest version — preserving the original UUID and version_number — then attempts an INSERT that violates the primary key and the `(document_id, version_number)` unique constraint. The fix adds an `update_approval` method to the `DocumentVersionRepository` port and its SQL implementation, and updates the router to call it instead of `create`.

**Primary Dependencies**: FastAPI (router), SQLAlchemy async (persistence), Pydantic v2 (entities), pytest + anyio (testing)

**Storage**: PostgreSQL — `document_versions` table, columns `approver_user_id` and `approved_at` already exist as nullable; no migration needed

**Testing**: pytest + anyio; contract tests with AsyncMock + patch pattern (existing file)

**Scale/Scope**: One endpoint (`POST /documents/{id}/publish`), one new port method, one new SQL UPDATE

### Phase C: Implement SQL adapter

File: `apps/api/tessera_api/adapters/repo.py`

Add to `SqlDocumentVersionRepository`:
```python
async def update_approval(
    self, version_id: UUID, approver_id: UUID, approved_at: datetime
) -> DocumentVersion:
    await self._session.execute(
        update(DocumentVersionModel)
        .where(DocumentVersionModel.id == version_id)
        .values(approver_user_id=approver_id, approved_at=approved_at)
    )
    result = await self._session.execute(
        select(DocumentVersionModel).where(DocumentVersionModel.id == version_id)
    )
    model = result.scalar_one()
    return _version_from_model(model)
```

Import `datetime` from `datetime` at top of file if not already present.

### Phase D: Update router

File: `apps/api/tessera_api/routers/documents.py`, function `publish_document`

