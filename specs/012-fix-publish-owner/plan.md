# Implementation Plan: Fix Document Publish — Auto-Assign Owner

**Branch**: `012-fix-publish-owner` | **Date**: 2026-06-19 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/012-fix-publish-owner/spec.md`

## Summary

Documents are created without `owner_user_id`, causing `POST /documents/{id}/publish` to return 400 "Document has no owner." The fix has three parts: (1) set `owner_user_id` at document creation time from the authenticated user's ID, (2) auto-assign the publisher as owner at publish time if the document has no owner (legacy fallback), and (3) fix `require_user` to include `"id"` in the JWT auth path so all endpoints can reliably access the user's UUID.

## Implementation Phases

### Phase A: Failing Tests (TDD — write before code)

Add to `apps/api/tests/contract/test_documents.py`:

1. `test_create_document_sets_owner_from_user_info` — verifies `Document(owner_user_id=<user_id>)` is passed to `doc_repo.create()`
2. `test_publish_document_succeeds_when_no_owner` — mocks a doc with `owner_user_id=None`, verifies publish succeeds and `doc_repo.set_owner` is called
3. `test_publish_document_preserves_existing_owner` — mocks a doc with `owner_user_id=<some_id>`, verifies publish succeeds and `doc_repo.set_owner` is NOT called

All three should **fail** before implementation changes.

### Phase B: Fix `require_user` JWT path

File: `apps/api/tessera_api/auth/oidc.py`

Add `"id": claims["sub"]` to the JWT branch return dict so `user_info["id"]` is consistent across session and JWT auth paths.

### Phase C: Fix `create_document` router

File: `apps/api/tessera_api/routers/documents.py`

Extract `owner_id = uuid.UUID(user_info["id"])` after `require_user`. Pass `owner_user_id=owner_id` to the `Document(...)` constructor.

### Phase D: Fix `publish_document` router

File: `apps/api/tessera_api/routers/documents.py`

Replace the 400 hard-rejection block for `doc.owner_user_id is None` with auto-assignment: call `assign_owner(doc, publisher_id)` + `await doc_repo.set_owner(document_id, publisher_id)` when owner is absent, then proceed to `lifecycle_publish`.
